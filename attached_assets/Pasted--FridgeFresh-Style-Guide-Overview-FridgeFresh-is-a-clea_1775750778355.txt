# FridgeFresh — Style Guide

## Overview

FridgeFresh is a clean, modern dinner suggestion app that helps users discover meals from ingredients they already have. The brand evokes freshness, simplicity, and confidence in the kitchen. The aesthetic is minimal and airy, inspired by contemporary wellness and food brands. The experience should feel calm and inviting — never cluttered or overwhelming.

**Brand Personality**: Fresh, approachable, trustworthy, effortless.

**Core Metaphor**: Your fridge, reimagined as a source of inspiration rather than stress.

## Iconography

**Style**: Outline/stroke icons, 1.5px stroke weight, rounded caps and joins. Use Lucide React (already in the project).

**Size tokens**:
- `icon-sm`: 16px — inline with text, chips
- `icon-md`: 20px — buttons, navigation
- `icon-lg`: 24px — section headers, empty states
- `icon-xl`: 40px — hero illustrations, feature highlights

**Key icons** (Lucide names):
- `Refrigerator` — fridge/home
- `ChefHat` — recipes/cooking
- `Plus` / `X` — add/remove ingredients
- `Search` — search
- `Clock` — cook time
- `Flame` — difficulty/heat
- `Utensils` — servings
- `Shuffle` — spin/suggest
- `Heart` — favourites
- `Leaf` — dietary tags (vegetarian/vegan)

**Colour**: Icons inherit `currentColor`. Default: Charcoal. Inside chips: Sage. Decorative: Warm Gray.

### Layout Rules

- **Max content width**: 480px (mobile-first, single column). Centered with auto margins.
- **Page padding**: `space-5` (20px) horizontal on mobile; `space-8` (32px) on tablet+.
- **Section spacing**: `space-10` (40px) between major sections.
- **Card padding**: `space-5` (20px) internal padding. Border radius: 16px. No visible border — use subtle shadow (`0 1px 3px rgba(0,0,0,0.06)`).
- **Ingredient chips**: Height 36px, padding `space-2` (8px) horizontal and `space-1` (4px) vertical, border-radius 9999px (pill), gap between chips `space-2` (8px). Background: Sage Light. Text: Sage. On remove: X icon at `icon-sm`.
- **Buttons**: Height 48px (primary), 40px (secondary). Border-radius 12px. Full-width on mobile for primary CTAs.
- **Recipe cards**: Vertical stack — image (aspect 16:9, border-radius 12px top), content below. Gap between cards: `space-4` (16px).

## Tone of Voice

- **Casual but not silly**. Friendly first-person plural ("Let's see what we can make").
- **Encouraging**: Never shame users for limited ingredients. ("Only 3 items? No problem — here's what you can whip up.")
- **Concise**: Short sentences. No jargon. Action-oriented labels ("Add ingredient", "Show me meals", "Spin again").
- **Warm**: Use food-related language naturally ("Fresh picks", "What's cooking", "Ready in 20 min").

## Component Patterns

### Ingredient Input Area
A large, prominent input field at the top with autocomplete suggestions. Added ingredients appear as pill-shaped chips below, wrapping naturally. Each chip has the ingredient name and a small X to remove.

### Suggestion/Spin Button
A single prominent CTA button below the ingredient area. Uses Sage as background. Includes the Shuffle icon. Subtle scale animation on press (scale 0.97 → 1.0, 150ms ease-out).

### Recipe Result Cards
Clean vertical cards. Top image with rounded corners. Below: recipe title (h3), cook time and difficulty as small icon+text pairs, and a row of matching ingredient badges (highlighting which of your fridge items are used). A heart icon for favouriting.

### Micro-interactions
- Chip add: fade-in + slide-up (200ms ease-out)
- Chip remove: fade-out + scale-down (150ms ease-in)
- Button press: scale 0.97 (100ms) then back
- Card appear: staggered fade-in (50ms delay per card, 300ms duration)
- Spin icon: 360deg rotation on click (500ms ease-in-out)

## Accessibility

- Minimum contrast ratio: 4.5:1 for body text, 3:1 for large text and icons.
- All interactive elements must have visible focus rings (2px Sage, 2px offset).
- Chips must be keyboard navigable (tab to focus, Enter/Backspace to remove).
- Screen reader announcements for ingredient add/remove and results loaded.
- Respect `prefers-reduced-motion`: disable all animations.
- Touch targets: minimum 44x44px.
